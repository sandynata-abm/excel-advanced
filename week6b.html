"Anda bekerja sebagai analis data di sebuah perusahaan retail multi-cabang. Manajer operasional ingin laporan penjualan yang bersih dan siap dianalisis, tetapi data datang dari beberapa sumber yang formatnya tidak seragam."

🧭 Langkah Awal Sebelum Mulai

  1. Buka Excel, lalu paste data dari tombol Salin Dataset ke sheet baru untuk setiap tabel.
  2. Blok data, lalu tekan Ctrl + T agar setiap source berubah menjadi Table.
  3. Pastikan setiap tabel punya nama yang jelas, misalnya Transaksi_Retail, Target_Cabang, dan Master_Produk.
  4. Buka tab Data, lalu pilih From Table/Range untuk masuk ke Power Query Editor.
  5. Periksa panel Applied Steps agar siswa memahami bahwa setiap aksi akan tercatat otomatis.

Tugas 1: Data Cleansing

Bersihkan data transaksi cabang agar siap diproses di Power Query Editor.

  1. Buka query Transaksi_Retail di Power Query Editor.
  2. Pilih kolom Nama_Pelanggan dan Cabang, lalu gunakan Transform > Format > Trim untuk menghapus spasi di depan dan belakang teks.
  3. Jika masih ada karakter tidak terlihat, lanjutkan dengan Transform > Format > Clean agar karakter non-printable ikut dibersihkan.
  4. Gunakan Replace Values pada kolom Cabang untuk mengganti data yang salah, misalnya Jakrta menjadi Jakarta dan JKT menjadi Jakarta.
  5. Periksa kolom Status_Pembayaran. Klik setiap nilai kosong lalu isi dengan Unknown menggunakan Replace Values atau Fill bila formatnya cocok.
  6. Urutkan data berdasarkan ID_Transaksi dan Tanggal, lalu pilih Home > Remove Rows > Remove Duplicates agar baris ganda terhapus.
  7. Klik ikon tipe data pada kolom Tanggal dan ubah menjadi Date.
  8. Ubah kolom Harga_Jual menjadi Decimal Number atau Whole Number sesuai kebutuhan pelaporan.
  9. Cek hasil akhir: pastikan nilai cabang sudah seragam, baris duplikat sudah hilang, dan tidak ada error tipe data.

Fokus tugas ini adalah memastikan data konsisten sebelum dianalisis. Urutan yang disarankan: Trim/Clean → Replace Values → Remove Duplicates → Change Type.

Tugas 2: Data Shaping

Ubah bentuk data target penjualan bulanan dari format lebar menjadi format yang lebih mudah dibaca sistem laporan.

  1. Buka query Target_Cabang_Bulanan di Power Query Editor.
  2. Pastikan kolom Kode_Cabang tetap menjadi identifier utama dan tidak ikut dipilih saat proses transformasi.
  3. Pilih semua kolom bulan, yaitu Jan sampai Des.
  4. Klik kanan pada kolom yang dipilih, lalu pilih Unpivot Columns agar data berubah dari format lebar ke format panjang.
  5. Ganti nama kolom Attribute menjadi Bulan supaya hasilnya mudah dipahami saat dipakai di PivotTable atau laporan.
  6. Ganti nama kolom Value menjadi Target_Penjualan agar makna angkanya jelas.
  7. Periksa apakah hasil akhir sudah menjadi tiga kolom inti: Kode_Cabang, Bulan, dan Target_Penjualan.
  8. Jika ada format angka yang berubah menjadi teks, ubah kembali tipe datanya sebelum Load ke worksheet.

Tugas ini melatih kemampuan shaping, yaitu mengubah struktur data agar sesuai kebutuhan analisis dan dashboard.

Tugas 3: Data Transforming

Transformasikan kolom mentah menjadi format bisnis yang lebih informatif.

  1. Buka query transaksi yang sudah dibersihkan pada tahap sebelumnya.
  2. Pilih kolom Kode_Produk, lalu gunakan Split Column berdasarkan delimiter tanda hubung agar terpisah menjadi bagian kode dan SKU.
  3. Buka kolom Tanggal lalu pastikan formatnya konsisten. Jika perlu, gunakan menu tipe data atau locale setting agar semua tanggal terbaca benar.
  4. Tambahkan kolom baru Jenis_Channel dari kolom Channel dengan logika sederhana: jika berisi Online, beri nilai E-Commerce; selain itu, beri nilai Offline.
  5. Untuk menghitung Margin, gabungkan dulu query transaksi dengan Master_Produk agar kolom Harga_Dasar ikut masuk ke tabel transaksi. Di Power Query, ini dilakukan dengan Home > Merge Queries, bukan dengan VLOOKUP.
  6. Setelah kolom Harga_Dasar muncul, klik Add Column > Custom Column dan buat rumus margin sederhana seperti [Harga_Jual] - [Harga_Dasar].
  7. Jika kolom Qty tersedia, gunakan rumus yang lebih tepat, misalnya [Harga_Jual] - ([Harga_Dasar] * [Qty]). Pada latihan ini, kolom Qty sudah disediakan supaya siswa bisa menghitung margin dengan asumsi jumlah unit per transaksi.
  8. Lakukan pengecekan hasil transformasi: nama kolom baru sudah terbentuk, nilai kategori sudah sesuai, margin terhitung, dan tidak ada baris yang berubah menjadi error.

Transformasi berarti mengubah isi data menjadi bentuk baru yang lebih berguna untuk analisis bisnis. Untuk margin, alurnya adalah Merge Queries ke master produk, ambil Harga_Dasar, lalu hitung dengan Custom Column.

🔗 Tugas 4: Data Combining

Gabungkan beberapa sumber data agar manajemen bisa melihat laporan penjualan yang lengkap dalam satu tabel final.

  1. Buka query transaksi final yang sudah melalui cleansing dan transforming.
  2. Klik Home > Merge Queries untuk menggabungkan data transaksi dengan tabel Master_Produk.
  3. Pilih kolom Kode_Produk pada kedua tabel sebagai key penghubung, lalu pilih jenis join yang sesuai, biasanya Left Outer.
  4. Setelah merge selesai, klik ikon expand pada kolom hasil merge untuk menampilkan Nama_Produk dan Kategori.
  5. Lakukan merge kedua dengan tabel referensi cabang berdasarkan Kode_Cabang agar nama cabang atau wilayah ikut masuk ke tabel final.
  6. Jika ada data transaksi dari periode berbeda, gunakan Append Queries untuk menyatukan baris dari tabel Januari, Februari, dan bulan lainnya.
  7. Periksa kembali kolom hasil akhir supaya tidak ada kolom penting yang hilang setelah proses kombinasi.
  8. Pastikan struktur final siap dipakai untuk analisis, misalnya berisi ID_Transaksi, Tanggal, Cabang, Produk, Kategori, Harga_Jual, dan Target_Penjualan.
